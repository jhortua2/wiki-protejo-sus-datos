Objetivos del trabajo

Objetivo General: Proponer una arquitectura de red adecuada para la empresa "Protejo sus Datos", evaluando las implicaciones técnicas y de seguridad de las diferentes opciones disponibles.
  • → Identificar el tipo de red que se implementaría con la propuesta inicial de los directivos.
  • → Evaluar las consecuencias de seguridad de una red entre iguales en el contexto dado.
  • → Describir las condiciones necesarias para implementar una red cliente-servidor.
  • → Comparar las ventajas y desventajas de la arquitectura cliente-servidor.
  • → Proponer una arquitectura de red completa con roles, sistemas operativos y perfiles de usuario.
Wiki / Red P2P

Red Peer-to-Peer

Propuesta inicial de los directivos · Conexión via switch compartido

La propuesta inicial consiste en conectar los computadores a través de un switch compartiendo únicamente el acceso a internet. Esto implementaría una red entre iguales (P2P), donde cada equipo actúa simultáneamente como cliente y servidor, sin un nodo centralizado.

Ventajas técnicas

💰
Bajo costo inicial
No requiere servidor dedicado ni licencias especializadas.
⚙️
Configuración simple
Basta con un switch y una conexión a internet.
👤
Pocos usuarios
Suficiente para un número muy reducido de usuarios con necesidades básicas.
🧑‍💼
Sin admin dedicado
No requiere administrador de red a tiempo completo.

¿Por qué no es suficiente?

Para una empresa cuyo modelo de negocio se fundamenta en la confidencialidad y la disponibilidad 24/7, esta arquitectura presenta riesgos críticos que la hacen inviable. Ver sección de Evaluación de Seguridad.

Sin control centralizado
Sin autenticación robusta
Sin auditoría
Vulnerable a malware
Wiki / Evaluación de Seguridad

Riesgos de la Red P2P

Consecuencias en materia de seguridad de la información

🔓
Ausencia de control centralizado
Sin un servidor que valide credenciales, cualquier equipo conectado al switch podría acceder a recursos compartidos sin autenticación robusta.
🕵️
Riesgo sobre la confidencialidad
Los ingenieros podrían acceder a información administrativa y viceversa, violando el principio de mínimo privilegio.
⏱️
Riesgo sobre la disponibilidad
Si un equipo falla o es comprometido por malware, puede afectar la conectividad del resto, comprometiendo la promesa del servicio 24/7.
📋
Ausencia de auditoría
No existen registros centralizados de quién accede a qué recurso ni cuándo. Imposible rastrear incidentes.
🦠
Vulnerabilidad ante ataques internos
En redes P2P, un equipo infectado puede propagar amenazas a todos los demás sin barreras de contención.
Conclusión de seguridad: Una red P2P es incompatible con el modelo de negocio de "Protejo sus Datos". La arquitectura expone datos sensibles de clientes a riesgos inaceptables, tanto internos como externos.
Wiki / Red Cliente-Servidor

Red Cliente-Servidor

Condiciones, ventajas y desventajas de la arquitectura recomendada

Condiciones para implementarla

  • 🖥️ Hardware servidor dedicado: Multiprocesamiento simétrico, alta RAM y almacenamiento redundante (RAID).
  • 💿 Sistema operativo de red: Windows Server, Linux (distro servidor) u otro SO especializado.
  • 🔌 Infraestructura de red: Switch gestionable, cableado Cat6+ y configuración de VLANs para segmentar tráfico.
  • 🌐 Definición de dominio: Todos los equipos clientes deben unirse al dominio administrado por el servidor.
  • 🔒 Política de seguridad: Grupos de usuarios, perfiles de acceso, contraseñas robustas y auditoría de eventos.
  • ☁️ Conectividad segura a la nube: Configuración de VPN o conexión dedicada para la gestión de servicios en Amazon Cloud.

Comparativa: Ventajas vs Desventajas

✅ Ventajas
  • + Control centralizado de acceso y credenciales.
  • + Políticas de dominio, cifrado y firewall centralizado.
  • + Escalabilidad horizontal y vertical.
  • + Backups centralizados, garantizando disponibilidad.
  • + Integridad de datos verificada en cliente, tránsito y almacenamiento.
  • + Reducción de costos operativos con servicios centralizados.
❌ Desventajas
  • − Costo inicial elevado: hardware, licencias y personal.
  • − Punto único de falla sin redundancia adecuada.
  • − Complejidad de administración y mantenimiento.
  • − Dependencia del servidor: si cae, los clientes quedan sin recursos.

Sistemas operativos sugeridos

Componente Sistema Operativo Función principal
Servidor Windows Server 2022 Ubuntu Server 22.04 Active Directory / Samba, gestión de dominio, autenticación, backups
Clientes Windows 11 Pro Ubuntu Desktop Estaciones de trabajo según perfil de usuario

Sistema de archivos y seguridad

  • 📁 NTFS en servidor Windows (descriptores de seguridad y marcas MACE para auditoría) o EXT4 en Linux.
  • 🔥 Firewall físico perimetral + Windows Defender / IPTables en equipos cliente.
  • 🔐 VPN para acceso remoto seguro a los servicios de Amazon desde cualquier ubicación.
  • 🔑 Autenticación de doble factor para acceso al servidor y a los servicios Cloud.
  • 🔒 Política de contraseñas robustas, gestionadas desde el dominio con caducidad periódica obligatoria.
Wiki / Grupos y Perfiles

Grupos de Usuarios

Perfiles de acceso según el rol dentro de la organización

Grupo Acceso permitido Restricciones
Administradores de sistema Acceso total al servidor, configuración de servicios y conexión con Amazon AWS. Ninguna.
Personal administrativo Carpetas de documentos, correo interno, internet. Sin acceso a consolas de administración.
Ingenieros de configuración Herramientas cloud, consolas AWS, repositorios de configuración. Sin acceso a datos administrativos.
Invitados / Soporte externo Solo internet. Sin acceso a ningún recurso interno.
Principio de mínimo privilegio: Cada usuario accede únicamente a los recursos estrictamente necesarios para su función, limitando el impacto potencial de un acceso no autorizado o compromiso de credenciales.
Wiki / Conclusiones

Conclusiones

Cierre del análisis y recomendaciones finales

Para una empresa como "Protejo sus Datos", la elección de la arquitectura de red no es un asunto menor ni exclusivamente económico. La arquitectura cliente-servidor, aunque más costosa en su implementación inicial, garantiza el control centralizado, la escalabilidad y las políticas de seguridad necesarias para cumplir con las promesas de confidencialidad y disponibilidad que el negocio exige.

Puntos clave

  • ✗ Implementar una red P2P significaría exponer datos sensibles a riesgos inaceptables de seguridad.
  • ✓ La arquitectura cliente-servidor permite centralizar el control de acceso y auditoría.
  • ✓ La segmentación por VLANs y grupos de usuarios garantiza el principio de mínimo privilegio.
  • ✓ La integración con Amazon AWS con VPN y 2FA asegura la capa cloud del servicio.
  • → Como ingenieros de sistemas, la mayor responsabilidad es proteger la información de quienes confían en los sistemas diseñados.
